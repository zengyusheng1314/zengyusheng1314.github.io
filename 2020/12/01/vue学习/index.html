插槽

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <div id="app">
        <todo-list>
            <!-- <template v-slot>    默认插槽   -->
            <todo-item @delete="handleDelete" v-for="item in list" :title="item.title" :del="item.del">
                <!-- vue.js 2.6语法  -->
                <template v-slot:pre-slot="{value}">
                    <!-- 本质上是返回组件的函数 可根据不同的值传递不同的内容 -->
                       <span>前置{{value}}</span>
                    </template>
                <template v-slot:suf-slot>
                        <span>前置</span>
                     </template>
                <!--  vue.js 2.5语法
                        <span slot="pre-slot"> 前置</span>
                    <span slot="suf-slot">后置</span> -->
            </todo-item>
            <!-- </template> -->
        </todo-list>
    </div>
    <!-- CDN 引入vue-->
    <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
    <script>
        // 创建一个组件
        Vue.component('todo-item', {
            // 属性声明
            props: {
                title: String,
                del: {
                    type: Boolean,
                    default: false, //默认值
                },
            },
            // 模板   插槽标签 slot
            template: ` 
             <li>
                <slot name="pre-slot"  :value="value"></slot>
                <span v-if="!del">{{title}}</span>
                <span v-else style="text-decoration: line-through">{{title}}</span>
                <slot name="suf-slot"></slot>
                <button v-show="!del" 
                @click="handleClick">删除</button>
            </li>
            `,
            data: function() {
                return {
                    value: Math.random()
                }
            },
            methods: {
                handleClick() {
                    console.log('点击删除按钮')
                    this.$emit('delete', this.title)
                }
            },
        })
        Vue.component('todo-list', {
            // 插槽标签 slot
            template: `
            <ul>
               <slot></slot>
        </ul>
            `,
            data: function() {
                return {
                    list: [{
                        title: '课程4',
                        del: false,
                    }, {
                        title: '课程5',
                        del: true,
                    }],
                }
            },

        })
        var vm = new Vue({
            el: '#app',
            data: {
                message: 'hello world',
                list: [{
                    title: '课程2',
                    del: false,
                }, {
                    title: '课程3',
                    del: true,
                }],
            },
            methods: {
                handleDelete(val) {
                    console.log('handleDelete', val)
                }
            }
        })
    </script>
</body>
</html>

双向绑定和单向数据流

<!-- 数据的双向绑定本质上是单向数据流
v-model 在内部为不同的输入元素使用不同的 property 并抛出不同的事件：
text 和 textarea 元素使用 value property 和 input 事件；
checkbox 和 radio 使用 checked property 和 change 事件；
select 字段将 value 作为 prop 并将 change 作为事件。
<template>
-->
  <div id="app">
        <TodoList>
           <input v-model="message">
           {{message}}
           <input :value="message" @input="handleChange">  
  </div>
</template>
<script>
export default {
  name: 'App',
  data() {
    return { }
  },
  methods: {
        handleChange(e){
            this.message=e.target.value
        }
  },
}
</script>

理解虚拟DOM及Key属性的作用

通过比对算法比对变化前DOM树和变化后的DOM树，减少DOM变化产生的性能消耗。

Key 的作用：提升Vue在操作DOM的效率

最好不要使用index作为Key,使用index作为Key在某些场景下会出现bug,例如对一个列表进行了插入和删除等等。

如果没有数据id，怎样生成key更符合规范呢？

如果没有唯一值，可以维护自增的id 只要保证静态唯一。

Vue这种间接操作DOM原理，性能方面比jquery的直接操作DOM性能更差一些？

的确要差一些，最终都是要操作DOM的，当然也分场景，但是框架是在性能和开发效率之间找到一个平衡点。

状态data vs 属性 props

  • 状态是组件自身的数据
  • 属性是来父组件的数据
  • 状态的改变未必会触发更新
  • 属性的改变未必会触发更新

合理应用计算属性和侦听器

计算属性 computed

  • 减少模板中计算逻辑

  • 数据缓存

  • 依赖固定的数据类型(响应式数据)

    <template>
  <div>
    <p>Reversed message1: "{{ reversedMessage1 }}"</p>
    <p>Reversed message2: "{{ reversedMessage2() }}"</p>
    <p>{{ now }}</p>
    <button @click="() => $forceUpdate()">forceUpdate</button>
    <br />
    <input v-model="message" />
  </div>
</template>
<script>
export default {
  data() {
    return {
      message: "hello vue"
    };
  },
  computed: {
    // 计算属性的 getter
    reversedMessage1: function() {
      console.log("执行reversedMessage1");
      return this.message
        .split("")
        .reverse()
        .join("");
    },
    now: function() {
      return Date.now();
    }
  },
  methods: {
    reversedMessage2: function() {
      console.log("执行reversedMessage2");
      return this.message
        .split("")
        .reverse()
        .join("");
    }
  }
};
</script>

    侦听器 watch

  • 更加灵活、通用

  • watch 中可以执行任何逻辑，如函数节流，Ajax异步获取数据，甚至操作DOM

    <template>
  <div>
    {{ $data }}
    <br />
    <button @click="() => (a += 1)">a+1</button>
  </div>
</template>
<script>
export default {
  data: function() {
    return {
      a: 1,
      b: { c: 2, d: 3 },
      e: {
        f: {
          g: 4
        }
      },
      h: []
    };
  },
  watch: {
    a: function(val, oldVal) {
      this.b.c += 1;
      console.log("new: %s, old: %s", val, oldVal);
    },
    "b.c": function(val, oldVal) {
      this.b.d += 1;
      console.log("new: %s, old: %s", val, oldVal);
    },
    "b.d": function(val, oldVal) {
      this.e.f.g += 1;
      console.log("new: %s, old: %s", val, oldVal);
    },
    e: {
      handler: function(val, oldVal) {
        this.h.push("😄");
        console.log("new: %s, old: %s", val, oldVal);
      },
      deep: true
    },
    h(val, oldVal) {
      console.log("new: %s, old: %s", val, oldVal);
    }
  }
};
</script>
