JavaSE语言基础

2020-11-27- 2020-12-24
后端-Java

语言基础

变量

分类

  1. 成员变量(Instance Variables) — 非静态字段(Non-Static Fields)

  2. 类变量(Class Variables)—静态字段(Static Fields)：使用static修饰符声明的字段； 无论类被实例化了多少次，类变量都只有一个副本。

  3. 本地变量；局部变量(Local variables)：将临时存储在方法内部的变量。

  4. 参数(Parameters)：为方法提供额外信息的变量。

命名

  • 变量名区分大小写，第一个字符只可以是大小写字母或美元符号“$”或下划线“_”。按照惯例，完全不使用美元符号字符，不建议变量名以“_”开头，不允许使用空格。
  • 后续字符可以是字母，数字，美元符号或下划线字符。约定为变量选择名称时，请使用完整的单词，而不要使用隐含的缩写，选择的名称不能是关键字或保留字。
  • 如果您选择的名称仅包含一个单词，则用所有小写字母拼写该单词(驼峰规则）。 如果它包含多个单词，请大写每个后续单词的第一个字母;变量存储了一个常量值，例如static final int NUM_GEARS = 6，则约定会稍有变化，将每个字母大写并用下划线字符分隔后续单词 。

基本数据类型

byte

8位的带符号的二进制补码，范围[-128,127]

short

16位的带符号的二进制补码整数，范围[-32768,32767]

int

32位的带符号的二进制的补码整数，范围[-2^31,2^31-1]

long

64位的带符号的二进制补码整数，范围[-2^63,2^63-1]

float

单精度32位IEEE 754浮点数，如果需要在大型浮点数数组中保存，请使用浮点数（而不是双精度数），双精度64位IEEE 754浮点数，此数据类型决不能用于精确值，例如货币

double

双精度64位IEEE 754浮点数，此数据类型决不能用于精确值，例如货币，而用BigDecimal

boolean

两个值:true,false,java规范中，没有明确指出值的大小,大小取决于JAVA虚拟机的实现

char

单个的16位Unicode字符，范围[0,65535]

默认值

声明字段时，不一定总是需要分配值。 编译器会将已声明但未初始化的字段设置为合理的默认值。 一般来说，此默认值将为零或null，具体取决于数据类型。 但是，通常认为依赖于此类默认值是不好的编程风格。

局部变量略有不同。 编译器永远不会将默认值分配给未初始化的局部变量。 如果您无法在声明它的地方初始化本地变量，请确保在尝试使用它之前为其分配一个值。 访问未初始化的局部变量将导致编译时错误。

String

字符串对象，它是不可变的，这意味着一旦创建，它的值就无法更改。

由final修饰的char数组

整型

如果整数文字以字母L或l结尾，则其类型为long； 否则为int类型。 建议您使用大写字母L，因为小写字母l很难与数字1区分开。byte，short，int和long的值可以从int文字中创建。

进制

十进制：其数字由0到9组成；

十六进制：其数字由数字0到9和字母A到F组成;

八进制：其数字由0到7组成；

二进制：其数字由数字0和1组成;

前缀0x表示十六进制，0b表示二进制，0表示八进制。

26(10进制) => 0x1a(16进制) => 032(8进制) => 0b11010(2进制)

浮点型

如果浮点数以字母F或f结尾，则其类型为float。 否则，其类型为double，并且可以选择以字母D或d结尾。

char和String

char和String类型的文字可以包含任何Unicode（UTF-16）字符。

char和String文字支持一些特殊的转义序列：\ b（退格），\ t（制表符），\ n（换行），\ f（换页），\ r（回车）， \“（双引号），'（单引号）和\（反斜杠）

类型转换

自动转换

通常是范围小的数赋值给范围大的数
byte->short- >int->long->float->double

强制转换

通常范围大的数赋值给范围小的数
double->float->long->int->short->byte
